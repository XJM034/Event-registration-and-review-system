# 管理员账号管理 - 超管权限编辑功能整合

## 📋 更新内容

### 1. 功能整合

将"设为超管/取消超管"按钮功能整合到"编辑"对话框中，简化操作流程。

### 2. 新增功能

#### 当前账号标识
- 在编辑对话框中显示"当前账号" Badge
- 使用 Shield 图标标识当前登录的管理员账号

#### 权限保护
- 不能修改当前登录账号的超管权限
- 当前账号的超管复选框自动禁用
- 显示警告提示："⚠️ 不能修改当前账号的权限"

### 3. UI 改进

**编辑对话框新增内容：**
- 姓名输入框
- 超级管理员复选框
- 当前账号 Badge（仅当前账号显示）
- 权限修改警告（仅当前账号显示）

**操作按钮简化：**
- 移除独立的"设为超管/取消超管"按钮
- 保留：编辑、重置密码、删除

## 🔧 技术实现

### 新增 API 端点

**`GET /api/admin/current`**
- 获取当前登录的管理员 ID 和权限
- 用于判断是否为当前账号

```typescript
{
  success: true,
  data: {
    id: string,
    is_super: boolean
  }
}
```

### 组件更新

**`edit-admin-dialog.tsx`**
- 新增 `is_super` 字段到表单状态
- 新增 `currentAdminId` 状态
- 新增 `isCurrentAdmin` 计算属性
- 添加超管复选框
- 添加当前账号 Badge
- 添加权限修改警告

**`admins-tab.tsx`**
- 移除 `handleToggleSuper` 函数
- 移除"设为超管/取消超管"按钮
- 简化操作按钮布局

### API 验证逻辑

**`PUT /api/admin/admins/[id]`**
- 支持同时更新 `name` 和 `is_super` 字段
- 验证：不能修改自己的权限（当 `is_super !== undefined`）
- 验证：不能取消最后一个超级管理员
- 同步更新 `admin_users` 表和 `auth.users.user_metadata`

## 📊 功能对比

### 更新前
- 独立的"设为超管/取消超管"按钮
- 编辑对话框只能修改姓名
- 无法识别当前登录账号
- 可能误操作修改自己的权限

### 更新后
- 超管权限整合到编辑对话框
- 一次操作可同时修改姓名和权限
- 清晰标识当前登录账号
- 自动禁用当前账号的权限修改

## 🎯 用户体验改进

### 操作流程简化
**之前**：编辑姓名 → 保存 → 点击"设为超管" → 确认
**现在**：编辑 → 修改姓名 + 勾选超管 → 保存

### 安全性提升
1. **视觉提示**：当前账号显示 Badge，一目了然
2. **操作限制**：当前账号的超管复选框自动禁用
3. **警告提示**：显示明确的警告信息
4. **后端验证**：API 层面双重保护

### 界面优化
- 减少一个操作按钮，界面更简洁
- 相关功能集中在一个对话框，逻辑更清晰
- 使用 Badge 和图标增强视觉识别

## 🔒 安全验证

### 前端验证
1. 检测当前登录账号 ID
2. 禁用当前账号的超管复选框
3. 显示警告提示

### 后端验证
1. 验证当前管理员身份
2. 检查是否修改自己的权限
3. 检查是否取消最后一个超管
4. 记录详细错误日志

## 💡 使用说明

### 编辑管理员信息
1. 点击管理员行的"编辑"按钮
2. 修改姓名（可选）
3. 勾选/取消"超级管理员"复选框（非当前账号）
4. 点击"保存"

### 当前账号标识
- 编辑对话框标题下方显示"当前账号" Badge
- 超管复选框自动禁用
- 显示警告："⚠️ 不能修改当前账号的权限"

### 权限限制
- 当前账号：只能修改姓名，不能修改权限
- 其他账号：可以修改姓名和权限
- 最后一个超管：不能取消超管权限
